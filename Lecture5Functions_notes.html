Lecture 5: Functions in C and C++

Introduction to Functions

Functions are fundamental to breaking down complex projects into smaller, more manageable units of code. They allow for code reusability, information hiding, minimizing exposure to internals, and reducing bugs. This kind of top-down design approach leads to code that is easier to understand, debug, and maintain.

What is a Function?

  • A function is a named block of code, carrying out a specific task.
  • It has a set of parameters it expects to receive.
  • It has a return type, which dictates the type of value it sends back to the caller.

Using Functions

You have been using functions such as printf. Understanding how to invoke or call these functions is crucial.

As programmers, we can create customized functions for our specific needs. The process involves defining a function prototype and later providing the function definition.

Function Prototype

A function prototype informs the compiler about:

  • The function's name.
  • The number and type of parameters.
  • The return type of the function.

Function Definition

When defining a function, in addition to the return type and name, you provide:

  • A set of parameter names and their types.
  • The body of the function, where the actual computations or operations are performed.

Function Call

A function call is where you invoke the function and pass any required arguments. The function performs its operation and returns a result, if applicable.

Information Hiding and Scope

Information hiding is a principle that ensures only the necessary details of a function's operation are visible to its caller, providing a clear public interface to the function. This is where a function's scope comes in:

  • A variable's scope indicates where the variable is accessible within the code.
  • Variables declared within a function are local to that function and are not accessible outside it.
  • Global variables, on the other hand, are accessible throughout the entirety of the code.

Parameter Passing

There are two main ways to pass arguments to functions:

  • Pass by value: A copy of the argument's value is passed to the function. Changes to the argument within the function do not affect the original variable.
  • Pass by reference: The actual variable's address is passed to the function, allowing changes within the function to affect the original variable.

Storage Classes

Variables can be categorized based on their storage duration and scope:

  • auto: The default storage class for local variables.
  • static: For retaining variable values even after the function ends. They can be local or global.
  • register: Hints to the compiler to store the variable in a register for fast access.
  • extern: Declares a global variable that can be accessed across multiple files.

Best Practices

Use local variables over global variables whenever possible.
